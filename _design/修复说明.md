# 修复说明 - 本地预览问题

## 问题原因

1. **_config.yml** 中使用了 `gems:` 配置项（已废弃），应改为 `plugins:`
2. **footer.html** 引用了原主题的一些依赖文件，但这些文件不存在：
   - share.js
   - geopattern.js
   - prism.js
3. 缺少 **search.json** 文件
4. 缺少 **favicon.ico**，改为使用 **favicon.svg**

## 已修复内容

### 1. 修复 _config.yml
- 将 `gems:` 改为 `plugins:`

### 2. 修复 footer.html
- 移除了不存在文件的引用
- 添加了可选脚本的动态加载逻辑

### 3. 创建 search.json
- 添加了文章搜索索引文件

### 4. 创建 favicon.svg
- 使用 SVG 格式的 favicon

## 重新预览步骤

1. **停止当前的 Jekyll 服务**（按 Ctrl+C）

2. **重新运行 Jekyll**
   ```bash
   cd /Users/suncan/myposts.github.io
   bundle exec jekyll serve
   ```

3. **访问网站**
   http://localhost:4000

## 可能的剩余问题

如果仍然有问题，请检查浏览器控制台（F12）查看具体的错误信息。

### 如果某些样式仍然不正确

可能需要清除浏览器缓存后刷新页面：
- Windows/Linux: Ctrl + F5
- Mac: Cmd + Shift + R
